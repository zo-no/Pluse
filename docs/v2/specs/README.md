# v2 Specs Index

## 目的

这个目录记录可以直接指导编码的实现 spec。

每份 spec 文档应回答：

- 这一次具体实现什么
- 涉及哪些模块、接口、数据或视图
- 验收标准是什么
- 这次不做什么

## 结构规则

- `specs/` 允许按 phase 拆分
- 一个大型 requirement 可以对应多个 spec
- spec 必须引用对应的 requirement 和 design
- 未确认的 spec 不进入编码阶段
