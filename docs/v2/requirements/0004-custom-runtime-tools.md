# Requirement 0004 — 自定义 Runtime 工具

## 背景

Pluse 目前内置了三个 AI 工具：Codex、Claude Code、MC。工具列表硬编码在后端，用户无法添加新工具。

随着社区出现更多 CLI AI 工具（如 opencode、aider 等），用户希望能将自己使用的工具接入 Pluse，享受任务调度、自动化等能力。

## 真实问题

用户使用的 AI 工具不止 Codex 和 Claude，但 Pluse 的工具选择列表是固定的，无法扩展。

## 目标

允许用户在设置页面自定义添加 Runtime 工具，添加后该工具出现在任务创建和任务详情的工具选择列表中，Pluse 可以用它执行任务。

## 成功标准

1. 用户可以在设置页面添加自定义工具（填写名称、命令、Runtime Family）
2. 自定义工具出现在任务创建 Modal 和 TaskDetail 的工具选择列表中
3. 配置持久化，重启后不丢失
4. 用户可以删除自定义工具

## 不在范围内

- 自动检测系统已安装的 AI 工具
- 自定义工具的模型列表配置（后续再加）
- 工具可用性检测（available 字段，后续再加）
