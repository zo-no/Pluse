# 0002 — Inbox Capture / 随手记

**状态**: draft
**类型**: requirement
**优先级**: medium

## 背景

用户在工作过程中会随时冒出新的想法、事项或后续动作。  
如果记录入口太重，用户要么中断当前工作去整理，要么干脆不记，导致信息丢失。

Pluse 需要提供一个极轻量的“随手记”入口，让用户可以在几秒内完成捕获，而不需要当场决定它属于哪个 Project、Quest 或执行流程。

## 顶层目标

这个需求的目标不是“增加一个新建按钮”，而是：

- 让用户在不中断当前工作的情况下完成快速捕获
- 让临时想法先进入统一收集箱，而不是立刻打散到各处
- 让后续整理、归类、转 Todo / Quest / 其他对象成为第二步，而不是第一步

## 问题定义

当前缺少一个足够轻的记录入口，会导致：

1. 用户在记录时被迫立刻做分类决策
2. 记录动作本身打断当前任务
3. 零散想法没有稳定的落点
4. 临时事项容易遗忘，或事后难以回收

## 核心需求

系统需要支持一种默认进入收集箱的快速捕获能力：

- 用户可以随时点击或触发一个入口
- 立即开启一次新的“随手记”记录
- 默认放入收集箱，而不是要求用户先选完整归属
- 记录过程应尽量轻，不要求填写复杂字段
- 后续可以再整理为 Todo、Quest、项目内事项或其他正式对象

## 不在范围内

当前需求文档不讨论：

- 收集箱最终的 UI 形态
- 随手记和 Todo / Quest 的最终映射模型
- 是否支持语音、截图、附件等输入形式
- 是否支持自动分类和自动归档

这些内容属于后续 design / spec 阶段。

## 成功状态

如果这个需求被满足，用户应能感受到：

- 冒出一个想法时，可以几秒内先记下
- 不需要当场决定归属和优先级
- 当前工作不会因为记录动作被明显打断
- 所有临时记录都有统一落点，后续可回收整理
