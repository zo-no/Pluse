# 0009 — Project Automation Closed Loop

**状态**: draft
**类型**: requirement
**优先级**: high

## 背景

Pluse 已经具备 Project、Quest、Run、Todo 和 Reminder 的基础能力，也已经开始把 `Quest(kind='task')` 在用户面收敛为「自动化」。

随着生活项目开始出现周期自动化，一个更大的问题变得清晰：

- 自动化不应只是更多定时任务
- 自动化不应把所有运行结果都推给用户
- 每个项目都需要形成自己的闭环
- 跨项目优先级需要单独治理，不能塞进某一个项目里

因此当前要解决的不是“再加几个自动化”，而是建立一套可逐步扩展的项目级自动化建设方法。

## 真实问题

当前如果继续直接增加自动化，会出现几类风险：

1. **项目边界失真**
   - 生活自动化如果开始管理财务、内容、研发，会让项目职责混在一起
   - 项目内事实、提醒和记录失去归属感

2. **信息透出失控**
   - 自动化可以高频运行，但人类注意力不能高频承接
   - 如果每次运行都产出提醒、待办或摘要，系统会从助手变成噪音源

3. **缺少渐进接入机制**
   - 不同项目的自动化成熟度不同
   - 没有一套从静默、摘要、提醒到强干预的升级路径

4. **UI 承载不足**
   - 右侧工作面板适合承接少量行动项
   - 多项目、多自动化、多运行状态需要独立的管理面，而不是继续堆在侧栏

5. **跨项目优先级没有控制平面**
   - 用户需要主线项目被优先触达
   - 但主线判断不应由任意一个项目的自动化自行决定

## 顶层目标

Pluse 需要支持一种渐进式项目自动化体系：

- 每个项目可以拥有自己的自动化闭环
- 自动化默认在项目内观察、判断、行动和记录
- 跨项目只通过独立的总控层管理注意力和主线优先级
- 新自动化先低透出运行，稳定后再升级提醒权限
- 用户只看到需要自己行动、判断或确认的少量高信号内容

## 核心需求

### 1. 项目内闭环

每个项目的自动化应默认只处理本项目内的事实和行动。

项目自动化的基本闭环是：

- 观察项目状态
- 判断是否偏航或阻塞
- 生成必要的项目内提醒
- 记录运行结果
- 为下一次运行留下可复用上下文

### 2. 跨项目总控与项目执行分离

跨项目总控只负责：

- 判断当前主线项目
- 调整提醒优先级
- 压缩非主线项目信息
- 汇总项目状态

跨项目总控不应替项目自动化执行项目内部工作。

### 3. 自动化需要成熟度等级

每个项目的自动化接入都应经历渐进过程，而不是一次性打开完整提醒能力。

最低需要区分：

- `L0 配置中`
- `L1 静默运行`
- `L2 摘要运行`
- `L3 提醒运行`
- `L4 强干预`

升级前需要确认该项目的自动化输出稳定、重复少、对用户有实际帮助。

### 4. 自动化透出必须受预算控制

自动化运行结果默认不进入用户注意力面。

只有满足以下条件时，才允许上浮：

- 需要用户行动
- 需要用户提供缺失信息
- 需要用户做关键判断
- 发生影响项目推进的异常
- 与主线项目直接相关

系统应支持按项目、按天、按主线优先级限制提醒透出量。

### 5. 新项目自动化需要接入计划

每个项目接入自动化前，都应先明确：

- 项目目标
- 闭环周期
- 自动化职责
- 允许透出的信息类型
- 默认提醒预算
- 不允许自动化做的事情

这份接入计划需要可维护，而不是散落在聊天记录里。

### 6. 自动化管理面需要升级

当自动化数量增多时，用户需要管理的是：

- 哪些项目有自动化
- 哪些自动化正在运行
- 哪些自动化失败或需要注意
- 哪些自动化会触达自己
- 下一次运行在什么时候

这不应继续完全依赖右侧工作面板。

## 不在范围内

当前 requirement 不规定：

- 具体数据库字段
- 具体 API / CLI 名称
- 自动化控制台的组件实现
- 是否拆独立服务或独立包
- 自动化 prompt 的最终模板
- 每个项目的完整自动化清单

这些内容属于后续 design / spec 或项目接入文档。

## 成功状态

如果这个需求被满足，用户应能感受到：

- 自动化是在帮每个项目形成闭环，而不是随机制造更多提醒
- 生活只管理生活，财务只管理财务，内容只管理内容
- 跨项目优先级有独立总控，不会被某个项目吞掉
- 新自动化先静默观察，再逐步升级
- 用户每天看到的提醒数量可控
- 自动化运行越多，用户看到的原始噪音越少
- 后续要接入新项目时，有清晰的接入顺序和升级标准

## 失败状态

如果后续实现出现以下情况，说明这个需求没有真正成立：

- 生活自动化开始替所有项目做总管
- 每个自动化运行后都产生可见提醒
- 项目之间的提醒和记录互相污染
- 自动化数量增加后，右侧栏变成运行日志瀑布
- 用户需要手工整理自动化产生的信息
- 新项目接入没有成熟度和透出预算，只是不断新增定时任务
