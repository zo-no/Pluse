# 0006 — 会话列表一键复制会话 ID

**状态**: draft
**类型**: requirement
**优先级**: medium

## 背景

在会话列表里给某个会话发消息前，用户经常需要先拿到 `quest.id`，目前往往要经过：

- 切到目标会话条目
- 手动选中 ID 文本
- 复制操作

这种流程对日常操作来说太繁琐，影响消息发送效率。

## 顶层目标

Pluse 需要在会话列表每条目内提供 **一个按钮**，让用户能直接复制该会话的 `id`，降低发消息前的手工步骤。

## 核心需求

### 1. 会话列表每条都能一键复制 ID

系统需要在会话列表每条目里提供一个 `Copy ID` 按钮，点击后复制该条目的 `quest.id` 到系统剪贴板。

### 2. 操作反馈可见

复制后需要有明确反馈（如成功提示/状态变化），让用户确认本次复制生效。

### 3. 复制结果可用于现有消息流程

复制的 ID 应可直接用于现有 CLI/Web 发消息流程（`quest message` 命令及详情页消息入口），不新增额外转换步骤。

## 成功状态

若该需求被满足，用户在处理会话消息时应能感受到：

- 每个会话项都能在当前列表视图内完成 ID 复制
- 复制动作为单步、低认知负担动作
- 在发消息前不需要再手工选中文本复制 ID

## 不在范围内

当前 requirement 不要求：

- 多会话批量复制
- 批量发消息链路设计
- 列表排序/筛选机制的重构
- 快捷键系统
- 命令行参数格式改造

## 讨论结果

该需求目前敲定为“列表内单会话单按钮复制”，不包含批量能力。
