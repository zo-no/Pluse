# 一次AI+人类共生体的生活方式探索

## 主线开头草稿 v1

很多 AI 产品解决的是“这一轮对话怎么更强”，但我更在意另一件事：
我们能不能在持续使用 AI 的过程中，慢慢长出自己的工作系统。

Pluse 想做的，不是再造一个聊天框，也不是只给你堆一层 skill、定时任务或者自动化按钮。它更像一个长期工作的容器：你和 AI 的对话可以继续，任务可以接上，项目可以沉淀，附件和运行记录可以留在原地，外部 CLI 和你自己的工具也可以一点点接进来。

这样一来，你积累的就不只是几段聊天记录，也不只是几个临时脚本，而是一套越来越像“你自己的系统”的工作方式。你今天只是让它帮你推进一个需求，明天可能把一个命令接进来，后天可能让一个 agent 去调用另一个 agent。很多东西不是一开始就设计好的，而是在使用过程中慢慢长出来的。

这也是我对 Pluse 最核心的期待：
它不只是替你完成任务，而是陪你把项目、工具、协作方式和上下文，一点点组织成一个可持续推进的系统。

### 宏大的思路

我想要把人类的各种内容都基于这个平台进行维护从抽象角度。

文件系统=人类工作的过程内容
会话=人类与AI的交互
任务=人类与AI的交互

这些加起来就是一个AI产品的最小雏形，理论上所有的AI产品都可以被拆成这样的结构。

由此All in one，把自己的数据统一记录，我相信未来会有神奇的事会发生。
